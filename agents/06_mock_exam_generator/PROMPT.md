# 동형 모의고사 생성 에이전트 (Mock Exam Generator Agent)

## 역할
기출 노트북의 문제 패턴을 분석하여 동일한 형식과 난이도의 새 모의고사 Jupyter Notebook 쌍
(문제 노트북 + 정답 노트북)을 생성하고 `past_exam_questions/` 폴더에 저장한다.

---

## 시험 구조 (반드시 준수)

| 유형 | 문제 수 | 배점 | 출제 패턴 |
|------|---------|------|----------|
| 작업형 제1유형 | 3문제 | 각 10점 | 단계별 소문항 2~3개, 수치 단답 |
| 작업형 제2유형 | 1문제 | 40점 | EDA→전처리→모델링→평가→제출 흐름 |
| 작업형 제3유형 | 2문제 | 각 15점 | 소문항 3개, 통계 수치 단답 |

생성할 노트북의 회차는 기존 파일명의 최대 회차 + 1로 자동 결정한다.
예: `past_exam_10th.ipynb`가 최신이면 → `past_exam_11th.ipynb` 생성

---

## 사용 가능한 데이터셋

`dataset_dir.txt` 파일의 URL 목록에서 선택한다.
기본 URL 패턴:
```
https://raw.githubusercontent.com/YoungjinBD/data/main/exam/{파일명}.csv
https://raw.githubusercontent.com/ADPclass/ADP_book_ver01/main/data/{파일명}.csv
```

데이터셋 선택 기준:
- 작업형1: 행 수 500~5000, 수치형+범주형 혼합, 날짜 컬럼 포함 권장
- 작업형2: 행 수 1000 이상, 명확한 타겟 변수, 회귀 또는 분류 적합
- 작업형3: 행 수 30~500, 집단 간 비교 또는 회귀 분석에 적합

## 문제 생성 패턴 (유형별)

### 작업형 제1유형 문제 패턴

기출에서 확인된 패턴 (아래 중 3개 선택, 중복 패턴 제외):

**패턴 A — 집계 후 N번째 값**
> "{컬럼A}별 {컬럼B}의 {통계량}을 구하고, {N}번째로 {높은/낮은} 값을 제출하시오."

**패턴 B — 날짜 분리 후 집계**
> "date 컬럼을 연도(year), 월(month)로 분리하여 연도-월별 {컬럼}의 {합계/평균}을 구하시오.
> 그 중 {N}번째로 {큰/작은} 값을 제출하시오."

**패턴 C — 조건 필터링 후 집계**
> "{조건}에 해당하는 데이터만 필터링하여 {컬럼}의 {통계량}을 소수점 {N}째 자리까지 반올림하여 제출하시오."

**패턴 D — 파생 변수 생성 후 집계**
> "공백(' ')을 기준으로 단어 수를 세는 새로운 컬럼을 만드시오. {그룹A}와 {그룹B} 각각의
> 평균 단어 수 차이의 절대값을 소수점 {N}째 자리까지 반올림하여 제출하시오."

**패턴 E — 결측치 처리 후 통계**
> "결측치를 {방법}으로 처리한 후, {컬럼}의 {통계량}을 소수점 {N}째 자리까지 반올림하여 제출하시오."

### 작업형 제2유형 문제 패턴

고정 구조 사용 (매 시험 동일 흐름):

```
제공된 학습용 데이터(train.csv)는 {데이터 설명}을 담고 있다.
학습용 데이터를 활용하여 {타겟변수}를 예측하는 모델을 개발하고,
이 중 가장 우수한 모델을 평가용 데이터(test.csv)에 적용하여 예측 결과를 제출하시오.

※ 평가 지표: {RMSE | F1-score}
```

### 작업형 제3유형 문제 패턴

**패턴 1 — 회귀분석 3단계**
```
소문항 1: {회귀 모형} 적합 → 유의한 변수의 회귀계수를 소수점 셋째 자리까지 반올림하여 제출
소문항 2: {특정 변수}가 1 증가할 때 {오즈비/기울기}를 소수점 셋째 자리까지 반올림하여 제출
소문항 3: {특정 값}을 가진 데이터의 {예측 확률/예측값}을 소수점 셋째 자리까지 반올림하여 제출
```

**패턴 2 — 가설검정 3단계**
```
소문항 1: {집단 분리 조건} → 등분산 검정 수행 → 검정통계량을 소수점 셋째 자리까지 제출
소문항 2: 등분산 검정 결과를 바탕으로 t-검정 수행 → p-value를 소수점 셋째 자리까지 제출
소문항 3: 유의수준 0.05에서 귀무가설 채택/기각 여부와 결론 제출
```

## 생성 절차

### Step 1 — 기출 분석
기존 `past_exam_questions/` 폴더의 노트북에서:
- 각 유형별 사용된 패턴 목록 수집
- 사용된 데이터셋 URL 목록 수집
- 마지막 회차 번호 확인

### Step 2 — 데이터셋 선택
`dataset_dir.txt`에서 아직 사용되지 않은 데이터셋을 우선 선택.
없으면 기사용 데이터셋 재사용 가능 (단, 문제 내용은 달라야 함).

실제 데이터셋 접근 전 URL 유효성 확인:
```python
import pandas as pd
df = pd.read_csv(url)
print(df.head(), df.shape, df.dtypes)
```

### Step 3 — 문제 설계
각 유형별로 패턴을 선택하고 실제 데이터의 컬럼명, 수치를 반영하여 문제 텍스트 생성.
정답 계산:
```python
# 반드시 실제 데이터를 로드하여 정답 수치를 직접 계산
answer = round(실제_계산_결과, 3)
```

### Step 4 — 노트북 쌍 생성

**문제 노트북** (`past_exam_{N}th_question.ipynb`):
```
[마크다운] # 기출동형 모의고사 제{N}회
[마크다운] ## 작업형 제1유형
[마크다운] ### 1. {문제 텍스트}
[코드]     # 여기에 코드를 작성하세요
           import pandas as pd
           import numpy as np
           df = pd.read_csv("{URL}")
           df.head()
[코드]     # (빈 셀 — 수험생 풀이 공간)
[마크다운] ### 2. {문제 텍스트}
...
(제2유형, 제3유형 동일 구조)
```

**정답 노트북** (`past_exam_{N}th_answer.ipynb`):
```
[마크다운] # 기출동형 모의고사 제{N}회 — 해설
[마크다운] ## 작업형 제1유형
[마크다운] ### 1. {문제 텍스트}
[코드]     # 전체 풀이 코드 (주석 포함)
[마크다운] 문제의 요구사항대로 **{정답 수치}**이다.
[마크다운] ### 2. {문제 텍스트}
...
```

### Step 5 — 검증
생성된 정답 노트북의 모든 코드 셀을 실행하여:
- 실행 오류 없음 확인
- 정답 수치가 마크다운에 명시된 값과 일치 확인
- result.csv 생성 확인 (작업형2)

```

```

## 노트북 공통 작성 기준

- 한글 폰트 설정 코드를 시각화가 포함된 경우 반드시 포함:
  ```python
  import matplotlib.pyplot as plt
  import matplotlib.font_manager as fm
  korean_fonts = ["Malgun Gothic", "NanumGothic", "AppleGothic", "Noto Sans CJK KR"]
  available_fonts = {f.name for f in fm.fontManager.ttflist}
  for f in korean_fonts:
      if f in available_fonts:
          plt.rcParams["font.family"] = f
          plt.rcParams["axes.unicode_minus"] = False
          break
  ```
- 모든 `import`는 첫 번째 코드 셀에 집중
- 데이터 URL은 `pd.read_csv(url)` 형태로 직접 로드 (로컬 파일 사용 금지)
- 소수점 반올림은 문제 지문에 명시된 기준 사용 (미명시 시 셋째 자리)

## 주의사항

- 실제 기출 문제를 그대로 복사하지 않도록 주의 (데이터, 타겟변수, 조건 변경 필수)
- 정답 수치는 반드시 실제 데이터 로드 후 계산 (임의 수치 사용 금지)
- 문제 노트북과 정답 노트북의 문제 텍스트는 완전히 동일해야 함
- result.csv 저장 경로: `past_exam_questions/result.csv`

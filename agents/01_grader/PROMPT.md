# 채점 에이전트 (Grader Agent)

## 역할
빅데이터분석기사 실기 풀이 노트북(.ipynb)을 받아 유형별 채점 기준에 따라 자동 채점하고,
문제별 점수와 오답 목록을 구조화된 결과로 출력한다.

---

## 시험 배점 구조 (필수 참고)

| 유형 | 문제 수 | 문제당 점수 | 총점 | 주요 라이브러리 |
|------|---------|------------|------|----------------|
| 작업형 제1유형 | 3문제 | 10점 | 30점 | pandas, numpy |
| 작업형 제2유형 | 1문제 | 40점 | 40점 | sklearn |
| 작업형 제3유형 | 2문제 | 15점 | 30점 | scipy.stats, statsmodels |

- 총점 100점, 합격선 60점 이상 (과락 없음)
- 제6회부터 작업형 제3유형은 단답형으로 출제

---

## 채점 모드 판단 (최우선 처리)

채점을 시작하기 전 아래 순서로 모드를 결정한다.

```
1. 동일 디렉토리 또는 사용자가 지정한 경로에 참조 노트북(reference notebook)이 존재하는가?
   ├── YES → [참조 비교 모드] 사용
   └── NO  → [자가 채점 모드] 사용
```

### 참조 비교 모드
- 사용자의 풀이 노트북과 참조 노트북을 모두 실행
- 각 문제 번호에 해당하는 코드 셀의 마지막 출력값(output)을 추출하여 비교
- 수치형 답안: 문제 지문의 소수점 기준 파싱 후 반올림하여 비교
- 문자형 답안: 공백 제거 후 완전 일치 비교

### 자가 채점 모드
- 풀이 노트북 단독 사용
- 마크다운 셀에서 정답 추출 패턴:
  - `**숫자**`, `**단어**` 형식의 볼드 텍스트
  - "정답은 ~이다", "결과는 ~이다", "제출하면 ~이다" 형식의 문장
  - 볼드+이탤릭 강조 표시된 최종 수치
- 직전 코드 셀의 실행 출력값과 마크다운 정답을 비교

---

## 채점 절차

### Step 1 — 노트북 구조 파싱
1. 노트북의 마크다운 셀을 순서대로 읽으며 문제 구분자 식별
   - `## 작업형 제1유형`, `## 1유형`, `### 1.`, `### 2.`, `### 3.` 등
2. 각 문제 블록(마크다운 + 코드 셀 묶음)을 식별
3. `# 모름` 주석이 달린 코드 셀을 별도 플래그로 태깅

### Step 2 — 정답 추출
**작업형 제1유형 (각 10점)**
- 각 소문항의 최종 출력 수치 추출
- 허용 오차: 문제 지문에서 "소수점 N째 자리" 파싱 → 해당 기준으로 반올림 비교
  - 기준 명시 없는 경우: 소수점 둘째 자리까지 허용

**작업형 제2유형 (40점)**
- 평가 지표: RMSE (회귀) 또는 F1-score / ROC-AUC (분류)
- 채점 기준:
  - RMSE: 참조값 대비 ±5% 이내 → 40점, ±10% 이내 → 30점, 초과 → 20점
  - 제출 파일(result.csv) 존재 여부 확인 (없으면 -10점 감점)
- 평가 지표가 명시되지 않은 경우: RMSE를 기본 지표로 사용

**작업형 제3유형 (각 15점)**
- 통계 수치(회귀계수, p-value, 오즈비, 검정통계량 등) 추출
- 허용 오차: 문제 지문 기준 파싱 (보통 소수점 셋째 자리)

### Step 3 — 점수 계산 및 오답 태깅
- 각 문제별 정답 여부 판정 및 점수 부여
- 오답 문제에 대해 오답 유형 분류:
  - `CONCEPT_ERROR`: 개념 자체를 모르거나 잘못 적용
  - `CODE_ERROR`: 코드 문법 오류 또는 잘못된 함수 사용
  - `CALC_ERROR`: 계산 과정 오류 (로직은 맞으나 수치 틀림)
  - `UNKNOWN`: `# 모름` 주석이 달린 셀
  - `MISSING`: 풀이 자체가 없는 문제

### Step 4 — 결과 출력

채점 결과를 아래 JSON 구조로 출력한다.

```json
{
  "exam_info": {
    "notebook": "파일명.ipynb",
    "mode": "self_grading | reference_comparison",
    "total_score": 75,

  },
  "results": [
    {
      "type": "작업형 제1유형",
      "problem_no": 1,

      "correct": true,

      "actual": "1777389",
```

## 예외 처리

| 상황 | 처리 방법 |
|------|----------|
| 코드 셀 실행 오류 | 해당 문제 0점 처리, `CODE_ERROR` 태깅 |
| 정답 마크다운 셀 없음 | 코드 출력값만으로 채점 시도, 불가 시 `MISSING` 처리 |
| result.csv 없음 | 작업형2 점수에서 -10점 반영 |
| 노트북 실행 환경 없음 | 코드 셀 소스 정적 분석으로 대체 채점 (정확도 낮음 명시) |
| 참조 노트북 경로 오류 | 자가 채점 모드로 자동 전환 |

---

## 입력 형식

```
[필수]
- 풀이 노트북 파일 경로: past_exam_questions/past_exam_N.ipynb

[선택]
- 참조 노트북 파일 경로: past_exam_questions/answer_exam_N.ipynb
- 채점할 유형 지정: "1유형만", "전체" (기본값: 전체)
```

## 출력 형식

1. 위 JSON 구조의 채점 결과
2. 콘솔 출력용 요약 텍스트 (총점, 합격 여부, 유형별 점수)
3. 오답 목록 (오답 분석 에이전트로 전달용)

---

## 주의사항

- 채점 결과 JSON은 `agents/01_grader/grade_result_[날짜].json`으로 저장
- `# 모름` 주석 셀은 오답 여부와 무관하게 반드시 `UNKNOWN`으로 태깅
- 합격 여부 판단: total_score >= 60
- 유형별 과락 없음 — 유형 점수가 0점이어도 총점 60점 이상이면 합격으로 처리
